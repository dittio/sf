Welcome to the Storyboard/Scene Microservices

The 'storyboard' currently support the following calls:

Service Discovery Functions

  1. / - gives this 'help' page
  2. /story/list - gives a list of all stories
  3. /story/<id> - gives the details of the storyboard with <id>. E.g.:
    /story/1 - to show the details of the first story in our database.

CRUD Functions

  1. CREATE - call /story with a request method of "post" (simulated by the form here with a 'name' and 'desc' payload, to add a story to the database with the given name and description.
  2. UPDATE - call /story/<id> with a request method of 'put', with a JSON payload with 'name' and 'desc' fields, to update the story specified by id, with the given name and description.
  3. DELETE - call /story/<id> with a request method of 'delete' to delete storyboard <id>, and all scenes. associated with it.

Test functions

  1. /story/add - this 'get' request (e.g. from the browser navigation bar) shows a form that can be used to simulate a JSON-based REST request to add a new story.
  2. /story/ed/<id> - this 'get' request (e.g.from the browser navigation bar) shows the story details in a form with an "update" and "delete" button that simulate submitting a JSON based "update" and "delete" request to the story microservice, for tests.
